# uScient Doctrine

## Core Identity

uScient is about user-owned knowledge and user-defined trust.

It exists to make personal context, source authority, privacy, and information boundaries inspectable, adjustable, portable, and governed by the user.

## The Central Problem

Modern systems increasingly ask users to share context before users can understand what they are sharing, where it goes, how it will be interpreted, and whether it can be revoked or constrained.

This creates a trust problem.

uScient starts from the position that trust should not be blindly granted. Trust should be manageable.

## The Central Claim

The user should define the rules for their own knowledge.

That means users should be able to inspect, label, route, restrict, export, revise, and revoke how their personal information and context are used.

## Agent Boundary

uScient is agent-compatible, not agent-run.

AI systems and agents may assist with review, labeling, summarization, routing, refactoring, auditing, and workflow execution, but they do not own the user's knowledge, define trust policy, or become the authority over personal context.

Agents are bounded consumers and operators.

The user defines the rules.  
The system exposes only permitted context.  
Sources retain authority.  
Boundaries remain inspectable.  
Actions must be reviewable.

Agentic workflows are acceptable only when they preserve user governance.

## What uScient Cares About

uScient cares about:

- user-owned knowledge
- user-defined trust
- manageable trust
- source authority
- personal information control
- inspectable boundaries
- evidence-based context handling
- local-first control where practical
- clear separation between private context and public claims
- AI assistance that increases user agency

## What uScient Is Not

uScient is not merely:

- privacy tools
- a password manager
- a note-taking app
- a generic AI wrapper
- an agent-run memory platform
- an autonomous personal knowledge agent
- a vague personal knowledge base
- a finished SaaS platform
- a promise that technology alone can solve trust

uScient may produce tools, documents, workflows, and systems, but those are expressions of the doctrine, not the doctrine itself.

## Public Maturity

uScient is early-stage.

Public materials should be honest about that.

Do not imply that uScient is already a mature platform, commercial product, compliance solution, or complete trust infrastructure.

## Doctrine Test

Every public statement about uScient should pass this test:

> Does this make uScient more precise, more trustworthy, and more distinct?

If not, revise it or remove it.
