# Language Style

uScient language should be precise, calm, and durable.

The writing should feel like it understands the stakes without exaggerating them.

## Voice

Preferred voice:

- clear
- exact
- restrained
- confident
- developer-readable
- public-safe
- doctrine-aligned

Avoid:

- hype
- filler
- vague futurism
- overclaiming
- performative urgency
- generic AI language
- inflated startup phrasing
- agent-run autonomy language

## Rhythm

Prefer short, strong sentences.

Use longer sentences only when they clarify relationships between ideas.

Good:

> Trust should be manageable.

Good:

> uScient is about user-owned knowledge and user-defined trust.

Good:

> Privacy matters, but uScient is not merely privacy tooling.

Good:

> Agents may operate within uScient boundaries. They do not set them.

Weak:

> uScient is a revolutionary AI-powered privacy platform that transforms how users interact with their personal data.

Weak:

> uScient uses autonomous agents to manage your knowledge for you.

## Preferred Phrases

Use:

- user-owned knowledge
- user-defined trust
- manageable trust
- personal information control
- source authority
- inspectable boundaries
- privacy/control
- evidence-based context handling
- local-first where practical
- governed by the user
- agent-compatible, not agent-run
- boundary-first AI
- reviewable action

## Use Carefully

Use only with context:

- AI
- platform
- product
- SaaS
- automation
- agent
- memory
- knowledge graph
- personal knowledge management
- data vault

These terms can distort uScient if they become the primary frame.

## Avoid

Avoid:

- revolutionary
- game-changing
- next-generation
- unlock your potential
- all-in-one
- trustless
- bulletproof privacy
- total control
- complete security
- fully private
- enterprise-ready
- the future of AI
- autonomous knowledge agent
- agent-run memory
- AI manages your knowledge
- your AI knows everything
- AI decides what matters

## Rewrite Rules

If a sentence sounds impressive but unclear, simplify it.

If a sentence makes a claim the project cannot yet prove, weaken or remove it.

If a sentence could describe dozens of AI startups, rewrite it.

If a sentence reduces uScient to privacy, expand it toward trust and user-owned knowledge.

If a sentence implies agents are in charge, rewrite it around user authority and inspectable boundaries.

If a sentence introduces a new term, define it or remove it.
