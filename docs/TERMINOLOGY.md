# uScient Terminology

This file defines preferred uScient language.

Use these terms consistently.

## uScient

Canonical spelling: `uScient`.

uScient centers the user's own knowledge and the user's authority over trust.

Avoid alternate spellings unless required by an external platform or identifier.

## User-Owned Knowledge

Knowledge, context, records, annotations, and interpretations that should remain under the user's authority.

This does not mean every byte must always live only on the user's machine. It means the user's knowledge should not be silently converted into platform-owned leverage.

## User-Defined Trust

Trust rules defined by the user.

The user should be able to decide which sources, tools, systems, people, and models are allowed to see, transform, retain, or act on specific context.

## Manageable Trust

Trust that can be inspected, adjusted, limited, revoked, and explained.

This is a core uScient phrase.

Use it when describing the larger goal.

## Personal Information Control

Practical control over how personal information is labeled, shared, summarized, routed, stored, exported, and withheld.

Do not reduce this to generic "privacy."

## Privacy / Control

Privacy is part of uScient, but it is not the whole frame.

Preferred phrasing:

> privacy/control

This keeps privacy tied to agency, not just secrecy.

## Source Authority

The ability to understand where information came from, why it should be trusted, what boundaries apply to it, and how strongly it should influence decisions.

## Context

Information selected for use in a specific situation.

Context is not neutral. It carries source, sensitivity, authority, and purpose.

## Boundary

A rule or limit around information access, interpretation, routing, retention, or action.

Boundaries should be inspectable and adjustable.

## Evidence-Based Context Handling

A practice of grounding AI and software behavior in inspectable sources, labels, rules, and traces rather than vague inference.

## AI Assistance

AI used as a bounded instrument that helps users inspect, label, summarize, redact, route, compare, draft, or audit context.

AI assistance should increase user agency.

It must not replace user authority.

## Agent Boundary

The rule that agents may operate within user-defined boundaries, but must not set those boundaries.

Preferred phrasing:

> Agents may operate within uScient boundaries. They do not set them.

## Agent-Compatible

Able to work with AI agents while preserving user governance, source authority, context minimization, and reviewable action.

Preferred phrasing:

> uScient is agent-compatible, not agent-run.

## Agent-Run

A discouraged framing.

Do not describe uScient as agent-run, AI-managed, autonomous over user knowledge, or governed by AI agents.

## Local-First

A design preference where practical.

Do not claim that all uScient work is strictly local-only. Prefer:

> local-first where practical

## Avoided Terms

Avoid using these as primary definitions for uScient:

- privacy tools
- AI platform
- agent platform
- personal knowledge management app
- autonomous memory system
- data vault
- SaaS platform
- productivity app
- second brain
- trustless system

These may describe adjacent concepts, but they do not define uScient.
